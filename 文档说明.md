## 基于 Spring Boot 3 + Vue3 的前后端分离 Java 快速开发框架（后端）

## 项目简介

Keystone 是一套开源、精简、可快速落地的全栈开发脚手架。本仓库为后端工程，适用于：

* 个人开发者快速搭建管理系统
* 团队内部中小型业务系统
* Spring Boot + DDD/CQRS 风格学习样例

本项目后端特性：

* 基于 Spring Boot 3.5.13
* 基于 Spring Security + JWT 的认证授权
* 三级缓存架构：Map → Caffeine（本地）→ Redis（分布式）
* MyBatis-Plus 数据访问
* 多数据源与数据权限控制
* 完整的单元测试与集成测试体系
* Gradle Configuration Cache + Build Cache，增量构建可低至 1s

前端仓库：<https://github.com/bruceblink/AgileBoot-Front-End>

## 快速开始

### 1. 环境要求

* JDK 25（推荐使用项目自带 Wrapper 构建）
* Gradle（推荐使用项目自带 Wrapper）
* MySQL 8+
* Redis 6+

> 建议使用项目自带 `gradlew` / `gradlew.bat`，避免本地版本差异。

### 2. 拉取代码

```bash
git clone https://github.com/bruceblink/Keystone
cd Keystone
```

### 3. 启动基础设施（推荐 Docker）

参考 [README.md](README.md) 中的 “Docker 开发环境” 章节

```bash
cd docker
docker compose up -d
```

数据库初始化脚本位于：

* [sql/mysql8/01_database.sql](sql/mysql8/01_database.sql) — 建库
* [sql/mysql8/02_agileboot-20230814.sql](sql/mysql8/02_agileboot-20230814.sql) — 基础数据
* [sql/mysql8/03_dict-20240101.sql](sql/mysql8/03_dict-20240101.sql) — 数据字典（v3.2.0 新增）

### 4. 配置环境

配置文件按职责拆分，**只需修改对应环境的 yml 文件**：

| 文件 | 职责 |
| :--- | :--- |
| [application.yml](keystone-admin/src/main/resources/application.yml) | 服务器端口 + 激活环境（`spring.profiles.active`） |
| [application-basic.yml](keystone-infrastructure/src/main/resources/application-basic.yml) | 所有公共配置（连接池、日志、Token、MyBatis、SpringDoc 等） |
| [application-dev.yml](keystone-admin/src/main/resources/application-dev.yml) | 开发环境：连接信息通过环境变量注入，默认值指向 localhost（Docker 暴露端口） |
| [application-prod.yml](keystone-admin/src/main/resources/application-prod.yml) | 生产环境：PostgreSQL + 生产 Redis，Swagger 关闭 |
| [application-test.yml](keystone-admin/src/main/resources/application-test.yml) | 单元测试：H2 内存库 |

**切换环境**：修改 `application.yml` 中的 `spring.profiles.active`，或通过环境变量覆盖：

```bash
# 本地开发（默认）
SPRING_PROFILES_ACTIVE=dev

# 生产环境
SPRING_PROFILES_ACTIVE=prod
```

> `spring.profiles.group` 已配置好，激活任意环境 profile 会自动附带 `basic`（公共配置），无需手动指定。

**开发环境**支持两种启动方式，均使用 `SPRING_PROFILES_ACTIVE=dev`：

| 方式 | 说明 |
| :--- | :--- |
| **本地 IDE + Docker 基础设施** | 先 `docker compose up -d mysql redis`，再在 IDE 中启动；连接默认指向 `localhost` |
| **全量 Docker** | `docker compose up -d`；`docker-compose.yml` 中的环境变量自动将连接地址覆盖为容器名 |

**本地 IDE 启动**只需修改 [application-dev.yml](keystone-admin/src/main/resources/application-dev.yml) 中的默认值，按需调整：

* 数据库连接（`SPRING_DATASOURCE_URL` / `SPRING_DATASOURCE_USERNAME` / `SPRING_DATASOURCE_PASSWORD`）
* Redis 连接（`SPRING_DATA_REDIS_HOST` / `SPRING_DATA_REDIS_PORT` / `SPRING_DATA_REDIS_PASSWORD`）
* 日志路径（`LOG_PATH`）

### 5. 构建与测试

```bash
./gradlew test
```

如果只验证构建：

```bash
./gradlew clean build -x test
```

> **构建性能提示**：项目已启用 Gradle Configuration Cache + Build Cache + 并行构建。
> 首次 `clean build` 约需 40–60s；配置未变更时复用缓存，后续增量构建可低至 1s 内完成。

### 6. 启动后端

启动类：`app.keystone.admin.KeystoneAdminApplication`

或直接打包运行：

```bash
./gradlew clean build -x test
java -jar keystone-admin/build/libs/keystone-admin.jar
```

## 无 MySQL / Redis 本地快速体验

切换到内置 H2 内存数据库（无需安装 MySQL / Redis）：

1. 修改 [application.yml](keystone-admin/src/main/resources/application.yml)，将 `spring.profiles.active` 设为 `test`
2. 启动项目，H2 控制台访问地址：`http://localhost:18080/h2-console`

```yaml
# application.yml
spring:
  profiles:
    active: test
```

> 注意：`test` 环境使用 H2 内存库，数据不持久化，仅适合本地体验与跑测试。

## Keylo 集成与用户注册流程

### 定位

Keystone 是企业管理后台入口，用户生命周期以 Keystone 为主；
Keylo 作为可选认证中心，负责外部身份认证与令牌签发。

### 当前登录模式

* 统一登录入口：`/login`
  * `auth.mode=local`：走 Keystone 本地账号认证。
  * `auth.mode=mixed` 且 `keylo.enabled=true`：Keystone 后端调用 Keylo 凭证校验接口完成认证，再映射本地用户并签发 Keystone 会话 token。
  * `auth.mode=keylo-only`：仅允许 Keylo 认证链路。
* Keylo token 交换登录：`/login/keylo`（兼容保留，推荐迁移到 `/login`）

Keylo 登录时，Keystone 通过 `sys_user.external_subject` 将 Keylo 主体映射到本地用户。

### 新增用户的注册链路（已实现）

在 Keystone 新增用户时，后端流程如下：

1. 先在 Keystone 落库创建用户。
2. 当 `keystone.auth.keylo.provisioning.enabled=true` 时，调用 Keylo 管理接口注册用户。
3. 先调用 Keylo 管理员令牌接口获取 `access_token`，再调用创建用户接口。
4. 从 Keylo 响应提取主体标识（默认使用 `uid`）。
5. 将主体标识回填到 `sys_user.external_subject`。

若 Keylo 注册失败，会抛出业务异常并中断新增流程（事务回滚）。

### 配置位置

环境相关配置在各环境 profile 中：

* `keystone-admin/src/main/resources/application-dev.yml`
* `keystone-admin/src/main/resources/application-prod.yml`
* `keystone-admin/src/main/resources/application-test.yml`

公共字段语义默认值在：

* `keystone-infrastructure/src/main/resources/application-basic.yml`

### 关键配置项

```yaml
keystone:
  auth:
    mode: ${KEYSTONE_AUTH_MODE:local}
    keylo:
      enabled: ${KEYSTONE_AUTH_KEYLO_ENABLED:false}
      issuer-uri: ${KEYLO_ISSUER_URI:http://127.0.0.1:2345}
      jwk-set-uri: ${KEYLO_JWK_SET_URI:http://127.0.0.1:2345/.well-known/jwks.json}
      credential-verify-url: ${KEYLO_CREDENTIAL_VERIFY_URL:http://127.0.0.1:2345/v1/auth/token}
      credential-me-url: ${KEYLO_CREDENTIAL_ME_URL:http://127.0.0.1:2345/v1/auth/me}
      provisioning:
        enabled: ${KEYLO_PROVISIONING_ENABLED:false}
        create-user-url: ${KEYLO_CREATE_USER_URL:http://127.0.0.1:2345/v1/admin/users}
        admin-token-url: ${KEYLO_ADMIN_TOKEN_URL:http://127.0.0.1:2345/v1/admin/token}
        admin-client-id: ${KEYLO_ADMIN_CLIENT_ID:cli-admin-root}
        admin-client-secret: ${KEYLO_ADMIN_CLIENT_SECRET:replace-with-strong-admin-secret}
```

### 配置建议（可选启用）

* 默认建议：`auth.mode=local`，`keylo.enabled=false`，`provisioning.enabled=false`。
* 需要接入 Keylo 登录时：先打开 `keylo.enabled=true`，再将 `auth.mode` 切到 `mixed`。
* 需要新增用户同步到 Keylo 时：再打开 `provisioning.enabled=true`。
* 若 Keylo 异常，可快速回切 `auth.mode=local`。

### 字段与错误码

* 用户映射字段：`sys_user.external_subject`
* 登录主体映射：默认使用 Keylo `uid` claim（可通过 `KEYLO_SUBJECT_CLAIM` 调整）
* 新增错误码：`Business.LOGIN_KEYLO_PROVISION_FAILED`
